# 首页杂志风格改造 — 设计文档

## 背景
用户反馈首页太单调，希望改为杂志/博客风格，增加视觉层次和内容引导。

## 设计目标
- 提升首页信息密度和视觉层次
- 引导用户发现内容（话题标签 → 精选 → 列表）
- 保持简洁，不增加复杂侧栏

## 改动清单

### HomePage.tsx
1. **Hero 区改造**
   - 保留品牌竖条 + 标题 + 副标题
   - 新增**话题标签栏**：从 `tagApi.list()` 取数据，横向滚动展示，点击跳转 `/articles?tag_id=X`
   - 标签栏下方分两栏：左为**精选文章大卡**（第一篇热门文章，16:9 封面大图 + 标题 + 摘要），右为**热门排行**（保留现有 `searchApi.hot`）

2. **内容区改造**
   - 移除右侧 `aside` 侧栏（含热门排行，已移至 Hero 区）
   - 纯双列文章网格 `grid sm:grid-cols-2`
   - 保留 `ArticleCard` 垂直卡片样式

### 后端改动（无）
全前端改动，不涉及 API 调整。

## 实现要点
- 话题标签栏使用 `overflow-x-auto scrollbar-thin` 横向滚动
- 精选大卡：取 `hotArticles[0]` 数据，用大号 `ArticleCard` 或手写布局
- 响应式：小屏标签栏可滑动，精选大卡和热门排行上下堆叠
- 无新增 API 请求，复用现有 `tagApi.list()` 和 `searchApi.hot()`
